# 字节 AI 芯片实习面试冲刺安排

## 基本判断
- 面试时间：`3月25日晚上`
- 每日可投入时间：`5-8小时`
- 当前最需要补强的方向：
  - `Gemmini/RSNCPU 项目深挖与表达`
  - `Transformer/MoE/Attention 与硬件映射`
  - `数字前端基础：AXI / CPU / CDC`
  - `代码实现：异步FIFO / 序列检测器`

## 总体策略
- 主线以 `Gemmini/RSNCPU 项目` 为核心，因为这是最容易拉开差距、也最容易被连续追问的部分。
- 次主线加入 `蜂鸟 E203 风格 RISC-V 处理器设计与验证`，因为这是你简历里最自然的 `CPU / 流水线 / 冒险 / 验证` 入口，面试官很可能据此追问你对顺序核、乱序核和微架构权衡的理解。
- `Transformer/MoE/Attention` 不再单独大面积铺开，而是作为项目背后的理论支撑来准备。
- `CPU / AXI / CDC` 作为底层防守模块准备，目标不是全覆盖，而是保证高频问题能讲清楚，并且能和你的蜂鸟项目形成闭环。
- 加入 `异步FIFO` 和 `序列检测器` 的代码实现训练，目的是同时提升：
  - RTL 手写能力
  - 状态机与 CDC 细节理解
  - “现场解释代码设计思路”的表达能力

## 时间分配
- `35%`：`Gemmini/RSNCPU 项目深挖与表达`
- `20%`：`CPU基础 + 蜂鸟项目深挖`
- `15%`：`Transformer/MoE/Attention 与硬件映射`
- `15%`：`AXI / CDC`
- `15%`：`异步FIFO / 序列检测器` 代码实现与验证
- `10%`：画图、口头模拟、复盘

## 每天固定节奏
1. `2-2.5小时`：项目深挖与表达
2. `1-1.5小时`：CPU基础 + 蜂鸟项目
3. `1-1.5小时`：Transformer/MoE/Attention 或 AXI / CDC
4. `1-1.5小时`：异步FIFO / 序列检测器代码实现
5. `0.5-1小时`：画图、录音模拟、复盘

## 代码实现专项要求

### 异步FIFO
目标不是“写完就算”，而是达到下面四层：
1. 能手写核心 RTL
2. 能解释为什么要 Gray code
3. 能解释满空判断
4. 能解释异步 FIFO 为什么能解决 CDC 问题

建议你准备的口头问题：
- 为什么单比特同步器不能直接处理多比特数据总线
- 异步 FIFO 里哪些信号需要跨域同步
- 为什么用 Gray code 而不是普通二进制计数
- full / empty 的判断依据是什么

### 序列检测器
目标：
1. 能独立写出状态机
2. 能解释 `Mealy` 和 `Moore` 的区别
3. 能处理重叠检测
4. 能从题意快速画出状态图

建议你准备的口头问题：
- 为什么这个题适合用有限状态机
- 重叠检测与非重叠检测如何区分
- 输出在状态上给还是转移上给
- 如果序列改成别的模式，状态设计如何变化

## CPU 与蜂鸟项目专项要求

### 蜂鸟 E203 风格 RISC-V 项目
目标不是只会复述“做过一个 CPU”，而是做到下面四层：
1. 能讲清两级流水主通路和关键模块边界
2. 能解释你写过或验证过的控制逻辑、访存路径和基础冒险处理
3. 能结合波形分析说明你如何定位 bug
4. 能从顺序核自然过渡到乱序核的理解

建议你准备的口头问题：
- 两级流水里最容易出问题的地方是什么
- 你怎么验证分支跳转、load/store 和基础冒险
- 如果出现访存相关 bug，你会怎么缩小定位范围
- 你理解的 `Tomasulo / ROB / 寄存器重命名` 分别解决什么问题
- 为什么乱序执行更强，但并不是所有核都做乱序
- 如果让你在现有顺序核上做性能增强，你会优先加什么

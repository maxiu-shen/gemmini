# 字节 IP 设计实习简历改写设计

> **路径约定**：文中涉及 `RSNCPU/`、`yolo_v11/`、`Internship/` 的路径以 Chipyard `generators/` 为根；Gemmini 源码与文档在 `gemmini/` 下。

**目标：** 基于既有实习简历、目标岗位要求以及仓库内 RSNCPU/Gemmini 真实材料，产出一版更贴近字节 `IP设计实习生` 岗位的中文简历改写稿，并在不失真的前提下适度前置未来 4 个月内高概率达到的阶段能力。

## 1. 岗位需求拆解

目标岗位重点关注以下能力：

- 芯片前端设计基础与 RTL 设计能力
- 对乱序 CPU、GPGPU 或 AI 计算架构方向的理解
- 能做矩阵/卷积计算结构探索、向量路线评估、数据流与带宽分析
- 主动学习、深入分析问题并形成工程落地方案

相比原简历，当前主要短板不在“做没做过 RTL/UVM”，而在“没有把现有经历组织成 AI 芯片架构/IP 设计叙事”。

## 2. 新简历的主叙事

新简历采用“双主线”写法：

1. **芯片前端/验证硬实力**
   - Verilog/SystemVerilog/UVM/AXI/RISC-V CPU/物理实现
2. **AI 芯片架构评估潜力**
   - Gemmini 源码理解
   - RSNCPU 可重构脉动阵列研究
   - YOLOv11n 在 Gemmini 上的算子映射与软硬件分工
   - 基于公开文献与真实工程约束建立性能/利用率/频率指标口径

这样既保留可信的工程基础，又能更贴合字节岗位对 AI 芯片设计评估的偏好。

## 3. RSNCPU / Gemmini 项目的改写原则

该项目作为简历第一项目，重点强调：

- 对 Gemmini 主链的系统理解：`RoCC -> LoopConv/LoopMatmul -> ReservationStation -> Controllers -> Scratchpad -> Mesh/PE`
- 对 YOLOv11n 在 Gemmini 上的算子拆分能力：哪些适合阵列加速，哪些应回退 CPU
- 对 RSNCPU 对标 SNCPU/Gemmini 的架构定位与指标分析能力
- 未来 4 个月可达的阶段成果，用“推进中 / 面向 / 建立基线 / 完成方案定义”等表述前置

避免写成：

- 已完成完整 RTL 实现
- 已拿到 FPGA 实测结果
- 已完成大模型/Transformer 全面优化

因为这些超出当前可信边界。

## 4. 能力放大边界

允许强化的内容：

- 将“源码阅读 + 分析笔记”升级表述为“建立 Gemmini 主链心智模型并形成可复用导读”
- 将“研究计划中的指标设计”升级表述为“建立架构评估口径与目标指标”
- 将“测试与部署准备”升级表述为“推进 bare-metal 部署基线与功能验证路径”
- 将“使用 AI 辅助脚本/日志处理”升级为“使用 AI 提升仿真、调试、源码理解与文档沉淀效率”

不写入简历的内容：

- 虚构 tapeout、流片、量产或 silicon bring-up 结果
- 虚构已完成 RSNCPU RTL/FPGA 全栈实现
- 虚构 BDD100K 最终训练指标或 Gemmini 端到端实测吞吐

## 5. 输出形式

本次交付生成一份项目内 `Markdown` 简历源稿，便于后续自行排版为 PDF：

- `shen_byte_resume_2026.md`

同时保留原有教育、项目、技能信息中的真实基础，但整体重排结构，提升岗位匹配度、招聘可读性和结果导向表达。
